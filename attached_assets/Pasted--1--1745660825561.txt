Техническое задание (ТЗ) для ИИ-агента: сервис поиска попутчиков в путешествиях
1. Общее описание
Сервис позволяет пользователям создавать маршруты (пешие, вело, авто, походы и др.) и находить попутчиков. Акцент — на комфортное путешествие в компании, без прямого давления на знакомства.

2. Функционал
2.1. Основные модули
Пользователи (регистрация, профиль, настройки).

Маршруты (создание, поиск, фильтры).

Чат (обсуждение поездки).

Рейтинги и отзывы (безопасность и доверие).

2.2. Детализация
🔹 Регистрация и профиль

Вход через email/телефон (+ OAuth: Google, VK).

Профиль: имя, фото, возраст, интересы (теги: #походы #городские_прогулки и т.д.).

Поле «Цель поездки» (туризм, друзья, романтика, компания).

🔹 Маршруты

Создание маршрута:

Тип (пеший, авто, вело, байдарки и т.д.).

Дата, точка старта и финиша (геолокация + карта).

Описание (например, «Ищу компанию для похода по Алтаю»).

Макс. кол-во участников.

Поиск маршрутов:

Фильтры: тип, дата, локация, рейтинг организатора.

Сортировка: по дате, популярности.

🔹 Чат

Личные сообщения (после принятия заявки).

Групповой чат для участников маршрута.

🔹 Рейтинги и безопасность

Отзывы после поездки (1-5 звёзд + текст).

Верификация:

Подтверждённый телефон.

Ручная проверка фото (опционально).

Жалобы и блокировка пользователей.

3. Технические требования
3.1. Стек технологий
Бекенд: Node.js (Express/Nest.js).

База данных: PostgreSQL.

Фронтенд: React (или просто адаптивный HTML/CSS для MVP).

Деплой: VPS (Ubuntu + Nginx) или облако (Render, Railway).

3.2. API-эндпоинты (примеры)
POST /auth/signup — регистрация.

POST /trips — создание маршрута.

GET /trips?type=hiking&date=2024-08-01 — поиск.

POST /chat/{trip_id} — отправить сообщение.

3.3. База данных (схема)
Таблицы:

users (id, name, email, verified, rating).

trips (id, user_id, type, date_from, date_to, location).

trip_members (trip_id, user_id, status: pending/accepted).

reviews (author_id, target_id, trip_id, rating, text).

4. Дополнительные нюансы
4.1. Безопасность
JWT-авторизация.

Проверка данных: запрет на дублирование маршрутов от одного пользователя.

Шифрование паролей (bcrypt).

4.2. UI/UX
Простой интерфейс: список маршрутов + карта (Google Maps API).

Уведомления: email/websocket о новых заявках.

4.3. Монетизация (на будущее)
Подписка PRO: приоритет в поиске, верификация.

Платные «тематические маршруты» (от партнёров).

5. Этапы разработки
MVP (2-3 недели):

Регистрация + создание маршрутов.

Базовый поиск.

Чат на WebSocket.

Вторая версия:

Рейтинги, отзывы.

Мобильное приложение (React Native).

Масштабирование:

Рекомендации (например, «похожие маршруты»).

Интеграция с Airbnb/Booking.

6. Проверка перед запуском
Тест на 20-30 пользователях (друзья, знакомые).

Исправление багов по фидбэку.
